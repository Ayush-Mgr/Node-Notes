## "zero probability" problem
If our model has never seen the exact phrase "love cats," it gives it a probability of 0, making the whole sentence "I love cats" seem impossible. This is brittle and obviously wrong

Smoothing takes a tiny bit of the probability "mass" from the N-grams we _did_ see (like "love dogs") and redistributes it to all the N-grams we _didn't_ see (like "love cats"). This ensures that no sequence ever has a probability of _exactly_ zero.

## Smoothing Techniques:

To handle unseen N-grams (zero probabilities), we use:

- Laplace (Add-1) Smoothing:
		
		P(wi|wi−1) = Count(wi−1,wi) + 1
		
		Count(wi−1) + V
	
- Backoff Models: Use lower-order N-gram if higher order is not found.
	
- Interpolation:
		
		P(wi|wi−2,wi−1) = λ1P(wi|wi−2,wi−1) + λ2P(wi|wi−1) + λ3P(wi)


### 1. Laplace (Add-1) Smoothing

This is the simplest way to solve the "zero probability" problem1.

The name **"Add-1"** tells you exactly what it does: We pretend we've seen _every single possible N-gram_ one more time than we actually did.

This changes our bigram formula.

Original Formula:

$P(w_i | w_{i-1}) = \frac{\text{Count}(w_{i-1}, w_i)}{\text{Count}(w_{i-1})}$

Laplace (Add-1) Formula:

$$P(w_i | w_{i-1}) = \frac{\text{Count}(w_{i-1}, w_i) + \mathbf{1}}{\text{Count}(w_{i-1}) + \mathbf{V}}$$



Let's break down the two parts of this change.

---

#### The Numerator: `Count(...) + 1`

This is the main fix.

- If we saw "love dogs" 5 times, its new count is $5 + 1 = 6$.
    
- If we _never_ saw "love cats," its new count is $0 + 1 = 1$.
    

**Problem solved.** The count is never zero, so the probability is never zero.

---

#### The Denominator: `Count(...) + V`

This is the part that balances the equation.

- **V** stands for **Vocabulary size**, which is the total number of unique words in our corpus3.
    

Why do we add V?

Think about it: for the context "love" (which is $w_{i-1}$), we didn't just add 1 to the numerator for "cats." We also added 1 for "dogs," 1 for "NLP," 1 for "teacher," and 1 for every other unique word in our vocabulary.

How many unique words are there? **V** of them.

So, since we added a total of **V** "imaginary" counts to the numerators, we must also add **V** to the total count in the denominator to make sure all the probabilities still add up to 1.

As your file notes, we are "effectively distributing '1 count' to every possible next word, and there are V of them"4.

The downside?

Laplace smoothing is simple but a bit crude. It takes a lot of probability "mass" away from things we know are common and gives it to things that might be impossible. It's often a starting point, but not the best-performing method.



#### Example


Let's use a tiny dataset.

- **Vocabulary (V):** 5 unique words $\rightarrow$ { "the", "cat", "sat", "on", "mat" }
    
- **Training Text:** "the cat sat"
    

We want to find the probability of the sentence "the cat on".

We need to calculate: $P(\text{"on"} | \text{"cat"})$

- **Original Formula:** $\frac{\text{Count}(\text{"cat on"})}{\text{Count}(\text{"cat"})}$
    
- **Counts:**
    
    - $\text{Count}(\text{"cat on"}) = 0$ (We never saw this pair)
        
    - $\text{Count}(\text{"cat"}) = 1$
        
- **Probability:** $\frac{0}{1} = 0$. The model says "the cat on" is impossible.
    

---

The Solution: Laplace (Add-1) Smoothing

We "add one" to every possible count.

- **New Formula:** $\frac{\text{Count}(\text{"cat on"}) + \mathbf{1}}{\text{Count}(\text{"cat"}) + \mathbf{V}}$
    
- **Calculation:**
    
    - **Numerator:** The count for "cat on" was 0. Now it's $0 + \mathbf{1} = \mathbf{1}$.
        
    - **Denominator:** The original count for "cat" was 1. We add **V** (the vocabulary size, which is 5). So, $1 + \mathbf{5} = \mathbf{6}$.
        
- **New Probability:** $\frac{1}{6}$
    

The sentence "the cat on" now has a small, non-zero probability.

 Summary: The "Why"

- Why add 1 (Numerator)?
    
    It's our fix. It's like adding one "imaginary" instance of "cat on" to our data, ensuring no count is ever zero.
    
- Why add V (Denominator)?
    
    This balances the math. We didn't just add an imaginary "cat on." We also added an imaginary "cat sat," "cat the," "cat cat," and "cat mat." Because our vocabulary (V) has 5 words, we added a total of 5 imaginary counts. To keep the probabilities fair and make them all sum to 1, our new total (the denominator) must be the Old Count (1) + Total Imaginary Counts (V=5).


Think of it like this:

**Original Counts (from "the cat sat"):**

- **Bigrams we saw:**
    
    - `Count("the cat")` = 1
        
    - `Count("cat sat")` = 1
        
- **Bigrams we _didn't_ see (examples):**
    
    - `Count("the sat")` = 0
        
    - `Count("cat on")` = 0
        
    - `Count("sat mat")` = 0
        
    - ...and 21 other possible pairs are also 0.
        

The "Processed Version" (Conceptual View)

After applying Add-1 smoothing, our new "conceptual" dataset has 5 new "imaginary" bigrams for _every_ word in the vocabulary.

Here's a look at just the counts for bigrams starting with **"cat"**:

| **Bigram**       | **Original Count**         | **Add-1 Smoothed Count**               |
| ---------------- | -------------------------- | -------------------------------------- |
| `("cat", "sat")` | 1                          | 2                                      |
| `("cat", "the")` | 0                          | 1                                      |
| `("cat", "cat")` | 0                          | 1                                      |
| `("cat", "on")`  | 0                          | 1                                      |
| `("cat", "mat")` | 0                          | 1                                      |
| **Total Counts** | ==**`Count("cat")` = 1**== | ==**`Count("cat") + V` = 1 + 5 = 6**== |
|                  |                            |                                        |

So, when we calculate a probability like $P(\text{"on"} | \text{"cat"})$, we use these new smoothed counts:

- **Numerator:** `Count("cat on")` is now **1** (not 0)
    
- **Denominator:** The total count for "cat" is now **6** (not 1)
    
- **New Probability:** $\frac{1}{6}$

###  Interpolation

The main idea is **blending**.

Instead of just relying on _one_ model (like a trigram), interpolation intelligently **combines the trigram, bigram, and unigram models** all at once.

#### How Interpolation Works

Think of it like getting advice. You don't just ask one expert (the trigram); you also ask a generalist (the bigram) and a beginner (the unigram) and then _blend_ their advice.


$$P(w_i | w_{i-2}, w_{i-1}) = \lambda_1 P(\text{trigram}) + \lambda_2 P(\text{bigram}) + \lambda_3 P(\text{unigram})$$

- The $\lambda$ (lambda) values are just weights that control how much you trust each model. (e.g., $\lambda_1=0.7, \lambda_2=0.2, \lambda_3=0.1$).
    
- All the weights must add up to 1 ($\lambda_1 + \lambda_2 + \lambda_3 = 1$).
