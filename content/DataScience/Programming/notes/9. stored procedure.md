It is a store block of code that could be used again and again just like functions




```SQL
delimiter //
create procedure <NAME>(
    IN <param1> <datatype>,     -- input only
    OUT <param2> <datatype>,    -- output only
    INOUT <param3> <datatype>   -- input & output
)

BEGIN
	<QUERY>......;
END//
delimiter ;

call <NAME>(<param1>);
```

- We use in perimeter when we are just passing the Value for reading

- For out and in-out, we need to give value that is stored in @variable because it changes the perimeter/input that we are giving to the procedure
	- the reason is @variable set a dedicated memory for that particular perimeter, which procedure could use to Store the processed value from the query, we sent the value without any @variable then there will be no place to store that processed value from the query
	-  `@Var-name` to set variable where var-name could be any name


## EXAMPLE:
Without parameters
```SQL
delimiter //
create procedure select_all()
BEGIN
	select  * from Sales_Table as a 
	Left join Product_Quality as b
	on a.product = b.product
    union 
	select  * 
	from Sales_Table as a 
	right join Product_Quality as b
	on a.product = b.product;
END //
delimiter ;

call select_all();
```

With perimeter

```SQL
delimiter //
create procedure select_all2(IN  var varchar(20))
BEGIN
	select  * from Sales_Table as a 
	Left join Product_Quality as b
	on a.product = b.product
    where a.product = var
    union 
	select  * 
	from Sales_Table as a 
	right join Product_Quality as b
	on a.product = b.product
    where a.product = var;
END //
delimiter ;


call select_all2('Apple');




```

```SQL
delimiter //
create procedure Total_product(OUT prod int)
begin 
select  count(*) into prod
from(
	SELECT a.product, a.price, b.quality
    from Sales_Table as a 
	Left join Product_Quality as b
	on a.product = b.product

    union 
    
	SELECT a.product, a.price, b.quality
	from Sales_Table as a 
	right join Product_Quality as b
	on a.product = b.product
    )AS combined;

end //

delimiter ;

drop procedure Total_product;

CALL Total_product(@prod);
SELECT @prod as Toal;
```

 __What is `@prod`?__

It’s not a column — it’s a **session variable**.

- Lives for your current database connection.
    
- Starts with `@` to tell MySQL “this is not a table field; it’s a variable in memory”.
    
- You don’t have to `DECLARE` it — MySQL creates it automatically when you use it.
