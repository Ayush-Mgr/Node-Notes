>**Single Inputs → Many Output = not Function.**

### **1. Concept Overview**

In Data Science, data rarely exists in isolation. We connect data points: Customers $\to$ Orders, or Inputs $\to$Predictions.

- **Relations** describe _any_ association between objects.
    
- **Functions** are specific, deterministic relations used for modeling.
    

### **3. Formal Definitions**

- Cartesian Product ($A \times B$): The set of all ordered pairs $(a, b)$ where $a \in A$ and $b \in B$.
    
    $$A \times B = \{(a, b) \mid a \in A, b \in B\}$$
    
    - Cardinality: $|A \times B| = |A| \cdot |B|$.
        
- Relation ($R$): Any subset of the Cartesian product.
    
    $$R \subseteq A \times B$$
    
- **Function ($f$):** A relation from $A$ to $B$ where **every** element in $A$ maps to **exactly one** element in $B$.
    
    - **Domain:** The set of all inputs ($A$).
        
    - **Codomain:** The set of all allowed outputs ($B$).
        
    - **Range (or Image):** The set of outputs that _actually_ occur. (Range $\subseteq$ Codomain).
        

### **4. Visual Representation**

### **5. Worked Examples**

**Example 1: Cartesian Product (Easy)**

- **Given:** $A = \{1, 2\}$, $B = \{a, b, c\}$.
    
- **Find:** $A \times B$ and its size.
    
- **Solution:**
    
    - Pair every number with every letter.
        
    - $\{(1,a), (1,b), (1,c), (2,a), (2,b), (2,c)\}$
        
    - Size: $2 \times 3 = 6$ elements.
        

**Example 2: Identifying Functions (Medium - Critical for Exam)**

- **Scenario:** Which of these are functions from $X=\{1, 2\}$ to $Y=\{a, b\}$?
    
    1. $R_1 = \{(1, a), (2, b)\}$
        
    2. $R_2 = \{(1, a), (1, b), (2, b)\}$
        
    3. $R_3 = \{(1, a)\}$
        
- **Solution:**
    
    1. **Function.** Every element in $X$ (1 and 2) has exactly one output.
        
    2. **Not a Function.** Input '1' maps to both 'a' and 'b'. (One-to-Many is forbidden).
        
    3. **Not a Function.** Input '2' has no mapping. (Every input must work).
        

**Example 3: Domain and Range (Challenging)**

- **Given:** Function $f: \mathbb{Z} \to \mathbb{Z}$ defined by $f(x) = x^2$.
    
- **Find:** Domain, Codomain, and Range.
    
- **Solution:**
    
    - **Domain:** $\mathbb{Z}$ (All integers: ..., -2, -1, 0, 1, 2, ...).
        
    - **Codomain:** $\mathbb{Z}$ (The definition says mapping to integers).
        
    - **Range:** The actual outputs. Squares are always non-negative.
        
    - Range = $\{0, 1, 4, 9, 16, ...\}$ (Perfect squares).
        
    - _Note: Range $\neq$ Codomain here._
        

### **6. Common Mistakes**

- **Confusing "One-to-Many" with "Many-to-One":**
    
    - **One-to-Many ($1 \to a, 1 \to b$):** BAD. Not a function.
        
    - **Many-to-One ($1 \to a, 2 \to a$):** GOOD. This is a valid function.
        
- **Range vs. Codomain:**
    
    - Codomain is the target space (e.g., "Real Numbers").
        
    - Range is where the arrows actually land (e.g., "Positive Real Numbers").
        

### **7. Data Science Connection**

- **Relational Databases:** A database table is literally a **Relation**. If you have a table with columns `(Student_ID, Course_ID)`, the rows are the subset of the Cartesian product of Students and Courses.
    
- **Machine Learning Models:** A classifier is a **Function**.
    
    - Input: Image ($x$).
        
    - Output: Label "Cat" ($y$).
        
    - We assume for one specific image, the "true" label is unique (deterministic).

# Types of functions 


---

### 1. Injective (One-to-One)

- **The Concept:** Uniqueness.
    
- **The Rule:** No sharing allowed. Every input gets its own unique output.
    
- **The Codomain (Target):** **Can** have leftovers (unused elements).
    
- **Analogy:** **Social Security Numbers.** Every person has a different number. No two people have the same number. But there are millions of possible numbers not yet assigned to anyone.
    <img src ='licensed-image 1.jpeg' style="filter: invert(1);">

### 2. Surjective (Onto)

- **The Concept:** Coverage.
    
- **The Rule:** No leftovers allowed in the target. Every element in the Codomain is hit by at least one input.
    
- **The Codomain (Target):** **Must** be fully used.
    
- **Analogy:** **Voting.** Every candidate (Codomain) receives at least one vote. It doesn't matter if one candidate gets 1 vote and another gets 50 (sharing is okay), as long as nobody gets 0 votes.
<img src ='licensed-image 2.jpeg' style="filter: invert(1);">



### 3. Bijective (One-to-One Correspondence)

- **The Concept:** Perfection (Injective + Surjective).
    
- **The Rule:** Unique partners only. No sharing, no leftovers anywhere.
    
- **The Codomain (Target):** **Must** be fully used AND everyone has exactly one partner.
    
- **Analogy:** **Monogamous Marriage.** (In a theoretical perfectly paired group). Every husband has exactly one wife, every wife has exactly one husband. No one is single, and no one has multiple spouses.

### Summary Checklist

|**Type**|**Leftovers in Target?**|**Sharing in Target?**|**Result**|
|---|---|---|---|
|**Injective**|Allowed|**BANNED**|Distinctness|
|**Surjective**|**BANNED**|Allowed|Full Coverage|
|**Bijective**|**BANNED**|**BANNED**|Perfect Reversibility|
