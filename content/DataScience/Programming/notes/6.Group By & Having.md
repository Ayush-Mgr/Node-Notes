## Group by
`GROUP BY` function in SQL is used to **group rows that have the same values in specified columns** into summary rows of [[0.1 Creating a DATABASE process#^ab5fb5|alias]] — like totals, averages, counts, etc. It's typically used with aggregate functions such as:

- `COUNT()`
    
- `SUM()`
    
- `AVG()`
    
- `MAX()`
    
- `MIN()`


```SQL
SELECT column1, aggregate_function(column2) AS <alias> 
FROM table_name
GROUP BY column1;
```
```SQL
select <col1> 
from <Table1>
Where <Coindition>
group by <col1>
order by <col>;
```


## Having

Having is same as where only difference is it could filter on the processed data 

```SQL
SELECT product, SUM(quantity) AS total_quantity
FROM Sales_Table
GROUP BY product
HAVING SUM(quantity) > 10;  -- filter groups after aggregation

```
or 
```sql
select sum(salary) as all_total_paid
from employe_salary
having all_total_paid>90;
```
### The "where vs. Having" Difference

1. **`WHERE` (The Raw Filter)**
    
    - Filters the raw rows **before** any math happens.
        
    - **Limitation:** It cannot see the future. It doesn't know what `SUM(salary)` is yet.
        
    - _Analogy:_ Checking ingredients at the supermarket. You throw away the bad apples _before_ you bake the pie.
        
2. **`HAVING` (The Cooked Filter)**
    
    - Filters the **Groups** (the results) **after** the math is done.
        
    - **Power:** It can filter based on totals, averages, and counts.
        
    - _Analogy:_ Taste-testing the pies. You throw away the whole pie if it tastes bad _after_ it is baked.
