### 1. What is Information Extraction?

**Information Extraction (IE)** is a sub-field of Natural Language Processing (NLP) that aims to automatically extract **structured information** (entities, relationships, events) from **unstructured or semi-structured text**.

### 2. Core Tasks in IE

#### 1. Named Entity Recognition (NER)

**Definition:** The task of locating and classifying named entities into predefined categories (Persons, Organizations, Locations, Dates, Monetary values, etc.).

**Example:**

- **Input:** _"Apple Inc. was founded by Steve Jobs in California in 1976."_
    
- **NER Output:**
    
    - **Apple Inc.** $\to$ Organization
        
    - **Steve Jobs** $\to$ Person
        
    - **California** $\to$ Location
        
    - **1976** $\to$ Date
        

**Approaches:**

- **Rule-based:** Regular expressions, gazetteers.
    
- **Statistical:** HMMs, CRFs.
    
- **Deep Learning:** BiLSTM-CRF, Transformer-based (BERT-NER).
    

#### 2. Relation Extraction (RE)

**Definition:** Identifying semantic relationships between two or more entities mentioned in a text.

**Example:**

- **Input:** _"Steve Jobs founded Apple Inc. in 1976."_
    
- **RE Output:**
    
    - **Relation:** founded
        
    - **Subject:** Steve Jobs (Person)
        
    - **Object:** Apple Inc. (Organization)
        
    - **Triplet:** (Steve Jobs, founded, Apple Inc.)
        

**Types of Relations:**

- **Directed:** Founder-of, Born-in, Works-at
    
- **Undirected:** Synonym-of, Related-to
    

#### 3. Event Extraction (EE)

**Definition:** Identifying real-world occurrences (events) from text along with participants, trigger words, time, location, and roles.

**Example:**

- **Input:** _"On 15 August 2023, a 6.5 magnitude earthquake struck Nepal, injuring dozens."_
    
- **EE Output:**
    
    - **Event:** Earthquake
        
    - **Trigger:** "struck"
        
    - **Date:** 15 August 2023
        
    - **Location:** Nepal
        
    - **Effect:** Injuring dozens
        

**Components:**

- **Trigger word:** Lexical signal (e.g., "launched", "attacked").
    
- **Event type:** Predefined class (e.g., Attack, Election).
    
- **Arguments:** Entities filling roles (e.g., Agent, Target).
    

### 3. Comparison of IE Tasks

|Feature|Named Entity Recognition (NER)|Relation Extraction (RE)|Event Extraction (EE)|
|---|---|---|---|
|**What it extracts**|Entities (people, orgs)|Relationships between entities|Real-world actions/events|
|**Input**|Raw text|Entities in context|Entities + triggers + context|
|**Output**|Tagged tokens|Triplets (Entity1, Relation, Entity2)|Structured events with roles|
|**Complexity**|Low to Medium|Medium to High|High|
|**Context Usage**|Moderate|High|Very High|
|**Application Domain**|General|KBs, QA, Networks|Journalism, Crisis monitoring|

### 4. Tools and Applications

|Stage|Tool Example|Application|
|---|---|---|
|**NER**|SpaCy, Flair, BERT-NER|Tag entities in news, legal docs|
|**Relation Extraction**|OpenNRE, Stanford RE|Build triplet databases|
|**Event Extraction**|DeepEventMine, ACE2005|Track disasters, financial news|

### 5. Challenges and Emerging Trends

**Challenges:**

1. **Ambiguity:** e.g., "Jordan" (Person or Country?), "Amazon" (River or Company?).
    
2. **Nested Structures:** Entity within entity, event within event.
    
3. **Cross-sentence relations:** Relationships spanning multiple sentences.
    
4. **Domain Adaptation:** Medical/Legal texts require specialized models.
    

**Emerging Trends:**

- **Joint Learning:** Combining NER, RE, and EE in one model.
    
- **Pretrained LMs:** Using BERT, SpanBERT, SciBERT.
    
- **Knowledge Injection:** Using external bases like Wikidata.
    
- **Cross-lingual IE:** Zero-shot models for low-resource languages.
