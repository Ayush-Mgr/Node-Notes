### 1. What is Question Classification?

**Definition:** Question classification is the process of identifying the **type** and **intent** of a natural language question. This allows a system to determine:

- The **expected answer type** (e.g., PERSON, DATE, PLACE).
    
- The best **processing strategy** (e.g., numeric QA vs. entity recognition).
    

### 2. Use Cases

|Application|Description|
|---|---|
|**Search Engines**|Improves answer precision by understanding what the user is looking for.|
|**Voice Assistants**|(Siri, Alexa) Classifies user intent to provide the expected answer format.|
|**Enterprise Chatbots**|Maps user queries to specific FAQ categories.|
|**Educational QA**|Understands the domain of the question (e.g., distinguishing History vs. Science).|

### 3. Techniques

#### A. Rule-based Classification

- **Method:** Uses regular expressions (regex) and keyword patterns.
    
- **Pros/Cons:** Lightweight but brittle.
    
- **Example:**
    
    ```
    if question.lower().startswith("who"):
        return "PERSON"
    ```
    

#### B. Statistical Classification

- **Models:** Naive Bayes, SVM, Decision Trees.
    
- **Requirements:** Requires **feature engineering** (n-grams, POS tags, "wh-" words).
    

#### C. Neural Classification

- **Models:** LSTM, BERT, DistilBERT.
    
- **Training:** Fine-tuned on datasets like **TREC**.
    
- **Advantage:** No manual feature engineering required; learns features automatically.

### 5. Architecture Example: BERT-based Classifier

**Scenario:** Inputting the question _"What is the capital of France?"_

**Process Flow:**

1. **Input:** `[CLS] What is the capital of France? [SEP]`
    
2. **Encoder:** Passed through **BERT Encoder** (12 layers).
    
3. **Embedding:** Extract the **[CLS] token embedding** (represents the whole sentence).
    
4. **Classification:** Passed through a **Dense Layer** $\to$ **Softmax**.
    
5. **Output Label:** `LOCATION` $\to$ `CITY`
    

**Training Details:**

- Fine-tuned using **cross-entropy loss**.
    
- Trained on question-label pairs from datasets like **TREC** or **CoQA**.
    

**Example Implication:**

- **Input:** _"When did World War 2 end?"_
    
- **Class:** `NUMERIC` $\to$ `DATE`
    
- **System Action:** The downstream system knows to specifically look for **date expressions** in the document corpus.
