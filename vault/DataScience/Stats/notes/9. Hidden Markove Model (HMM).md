
>**HMMs** are built for **labeling** or **tagging**.
> Tagging the important factor from the data set that affects the outcome, and  we cannot evaluate or see important factor directly 

they're statistical models assuming there's a sequence of **hidden states** we can't see, and these states _generate_ the **observations** (the words) we _can_ see.


HMM is used for:

• Part-of-Speech (POS) tagging
• Named Entity Recognition
• Speech recognition
• Spelling Correction


example for **Part-of-Speech (POS) tagging**.

- The **hidden state** is the tag (e.g., 'Noun', 'Verb').
    
- The **observation** is the actual word (e.g., 'dogs', 'bark')

HMM is  made of 5 components:

1. Hidden state ( S ): unobservable state that model moves between.
2. observation ( V ): observable states that each hidden state generates.
3. **Transition Probability (A)** ➡️: This is the probability of moving from one _hidden state_ to another _hidden state_ , _the probability is calculated through external data set and our input does not affect it. It is a ==rule== that is meant to follow to get our  hidden state_.
    
    - _Question it answers:_ "If I'm in state 'Noun', what's the chance the _next_ state is 'Verb'?" 
        
    - _Example:_ $P(\text{Verb} | \text{Noun})$
        
4. **Emission Probability (B)** 🎤: This is the probability that a _hidden state_ will "emit" (or produce) a specific _observation_, _the probability is calculated through external data set and our input does not affect it. It is a ==rule== that is meant to follow to get our  observation_.
    
    - _Question it answers:_ "If I'm in state 'Noun', what's the chance the word I see is 'dogs'?" 
        
    - _Example:_ $P(\text{dogs} | \text{Noun})$
        
5. **Initial State Distribution ($\pi$)** : This is the probability that a sentence _starts_ with a particular hidden state, _the probability is calculated through external data set and our input does not affect it. It is a ==rule== that is meant to follow to get our P(first hidden state)_.
    
    - _Question it answers:_ "What's the chance the _very first_ tag in a sentence is 'Noun'?" 
        
    - _Example:_ $P(\text{Noun is the first state})$
        

These five components (S, V, A, B, $\pi$) are all the model needs.


## HMM Language Modeling Example

Purpose: To show how HMM can model the relationship between observed words and
their syntactic or hidden categories (e.g., POS tags).
Given:
• Hidden states: S = {Noun (N), Verb (V)}
• Vocabulary: V = {dogs, bark}

• Transition probabilities:

P(N | N) = 0.1, P(V | N) = 0.9, P(N | V) = 0.8,  P(V | V) = 0.2

• Emission probabilities:

P(dogs | N) = 0.6, P(bark | N) = 0.4P(dogs | V) = 0.1, P(bark | V) = 0.9

• Initial probabilities:

π(N) = 0.7,  π(V) = 0.3

Task 1:  probability of How likely is it to start with 'Noun' _AND_ be "Run"?
	
- Score = π(Noun)×P("Run"∣Noun)
    
- Score = 0.7×0.01=0.007

Task 2: Calculate the probability that the sentence “dogs bark” was generated through
the hidden state sequence “Noun Verb”.

Step-by-step:
	
1. P(dogs, bark, N, V) 
	
2. π(N) · P(dogs | N) · P(V | N) · P(bark | V)
	
3.  0.7 · 0.6 · 0.9 · 0.9 
	
4. 0.340


## PROBLEMS

1. **Evaluation Problem (Forward Algorithm):** 
	- In the above example, the observations as well as the hidden states with Path are presented P(V,S) , but but what if only the observations are given which arise the problem of ambiguity ("dogs" could be a Noun or a Verb), we cannot be 100% sure of the hidden path., and we could not choose the best performing part due to in some cases two paths being 50-50 probability .
	- Solution,$$P(O) = \sum_{\text{all possible } Q} P(O, Q)$$
		- Sum of Probability of every paths$$P(V) = P(V, S_1) + P(V, S_2) + P(V, S_3) + \dots$$
    
2. **Decoding Problem (Viterbi Algorithm):** given that model has  very long sentence and brute, forcing it will cost a lot of time complixity . To fix that we use [[Viterbi Algorithm]] which significantly reduces the time complexity.
    
3. **Learning Problem (Baum-Welch Algorithm):** Given _only_ a bunch of sentences (observations), how can we _learn_the best rules (A,B,π) for the model? .
    


## HMM Algo

The Hidden Markov Model (HMM) algorithm can be implemented using the following steps:

- **Step 1: Define the state space and observation space:** The state space is the set of all possible hidden states, and the observation space is the set of all possible observations.
- **Step 2**: **Define the initial state distribution:** This is the probability distribution over the initial state.
- **Step 3: Define the state transition probabilities:** These are the probabilities of transitioning from one state to another. This forms the transition matrix, which describes the probability of moving from one state to another.
- **Step 4: Define the observation likelihoods:** These are the probabilities of generating each observation from each state. This forms the emission matrix, which describes the probability of generating each observation from each state.
- **Step 5: Train the model:** The parameters of the state transition probabilities and the observation likelihoods are estimated using the Baum-Welch algorithm, or the forward-backward algorithm. This is done by  iteratively updating the parameters until convergence.
- **Step 6: Decode the most likely sequence of hidden states:** Given the observed data, the Viterbi algorithm is used to compute the most likely sequence of hidden states. This can be used to predict future observations, classify sequences, or detect patterns in sequential data.
- **Step 7: Evaluate the model:** The performance of the HMM can be evaluated using various metrics, such as accuracy, precision, recall, or F1 score.
