**First-Order Logic (FOL)** (also known as Predicate Logic) extends propositional logic by including quantifiers, predicates, variables, and constants. Here's a basic guide to writing and understanding FOL, along with its symbols:

---

### **Basic Components of First-Order Logic**
1. **Constants**  
   Represent specific objects in the domain.  
   **Example:** `John`, `5`, `Paris`

2. **Variables**  
   Represent general elements in the domain.  
   **Example:** `x`, `y`, `z`

3. **Predicates**  
   Represent properties or relationships between objects.  
   **Example:**  
   - `Likes(John, Pizza)` means "John likes pizza."
   - `GreaterThan(x, y)` means "x is greater than y."

4. **Functions**  
   Represent mappings from objects to other objects.  
   **Example:**  
   - `Mother(x)` returns the mother of `x`.  
   - `Square(3)` returns `9`.

5. **[[1 . Propositional logic|Logical Connectives]]**  
   Combine predicates or formulas:
   - **¬ (Negation):** "not"
   - **∧ (Conjunction):** "and"
   - **∨ (Disjunction):** "or"
   - **→ (Implication):** "if-then"
   - **↔ (Biconditional):** "if and only if"

6. **Quantifiers**  
   Extend statements over all or some objects in the domain:
   - **Universal Quantifier (∀):** "for all"  
     **Example:** `∀x Likes(x, Pizza)` → "Everyone likes pizza."
   - **Existential Quantifier (∃):** "there exists"  
     **Example:** `∃x Likes(x, Pizza)` → "There is someone who likes pizza."

---

### **Steps to Write Predicate Logic**
1. **Identify the Domain:**  
   The set of all objects being discussed.  
   Example: If talking about people, the domain is "all people."

2. **Define Predicates:**  
   Describe properties or relationships.  
   Example: `Likes(x, y)` → "x likes y."

3. **Use Quantifiers:**  
   Specify whether statements apply to all objects or some objects.  

4. **Combine with [[1 . Propositional logic|Connectives]]:**  
   Form complex logical statements using `∧`, `∨`, `→`, `↔`, etc.



---

### **Common Symbols in Predicate Logic**

| **Symbol** | **Meaning**          | **Example**                        |
| ---------- | -------------------- | ---------------------------------- |
| ∀          | For all              | `∀x Likes(x, Pizza)`               |
| ∃          | There exists         | `∃x Likes(x, Pizza)`               |
| ¬          | Not                  | `¬Likes(John, Pizza)`              |
| ∧          | And                  | `Happy(John) ∧ Likes(John, Pizza)` |
| ∨          | Or                   | `Happy(John) ∨ Likes(John, Pizza)` |
| →          | Implies (If-Then)    | `Likes(John, Pizza) → Happy(John)` |
| ↔          | If and Only If (Iff) | `Happy(John) ↔ Likes(John, Pizza)` |

---

## EXAMPLE
The statement **"All that glitters is not gold"** can be translated into **First-Order Logic (FOL)**. Let's break it down step by step:

---

### **Key Insights**
1. **Domain of Discourse**:  
   Objects that "glitter."

2. **Predicate Definitions**:  
   - `Glitters(x)` → "x glitters"  
   - `Gold(x)` → "x is gold"

3. **Meaning of the Statement**:  
   Not everything that glitters is gold.  
   This implies:  
   - **There exists something that glitters but is not gold.**

---

### **FOL Representation**
Using existential quantifiers and logical connectives, the statement can be expressed as:  

$$
\exists x (Glitters(x) \land \neg Gold(x))
$$

---

### **Explanation**
- `∃x` → "There exists some object x"  
- `Glitters(x)` → "x glitters"  
- `∧` → "and"  
- `¬Gold(x)` → "x is not gold"

So, the whole statement reads:  
**"There exists an x such that x glitters and x is not gold."**
