
A **window function** performs a calculation **across a set of rows related to the current row** — **without collapsing them into one row** like `GROUP BY` does.

> [!NOTE]  
> The “window” is simply _the set of rows_ the function looks at when calculating a value for each row.

---

## **1️⃣ Key Concept**

- **Normal Aggregates (`SUM`, `AVG`, etc.)**  
    Group rows together and return **one row per group**.
    
- **Window Functions (`... OVER(...)`)**  
    Calculate a value across related rows, but **keep every original row** in the output — the calculated value appears **next to** each row.
    

---

## **2️⃣ Basic Syntax**

```sql
function_name (expression) 
OVER (
    [PARTITION BY partition_expression]  -- optional grouping
    [ORDER BY sort_expression]           -- optional sorting
)
```

---

## **3️⃣ Examples**

### **No Partition**

```sql
SELECT 
    product,
    price,
    SUM(price) OVER() AS total_price
FROM Sales_Table;
```

**Meaning:** Calculate the sum of all `price` values in the whole table, and display it in every row.

---

### **With Partition**

```sql
SELECT 
    product,
    price,
    SUM(price) OVER(PARTITION BY product) AS total_price_per_product
FROM Sales_Table;
```

**Meaning:** Split the table into groups by `product`, sum prices within each group, and show the result on each row in that group.

---

### **With Partition + Order**

```sql
SELECT 
    product,
    price,
    SUM(price) OVER(PARTITION BY product ORDER BY price) AS running_total
FROM Sales_Table;
```

**Meaning:** Within each product group, sort by `price` and compute a running total.

---

## **4️⃣ Common Window Functions**

- **Ranking**: `ROW_NUMBER()`, `RANK()`, `DENSE_RANK()`
    
- **Aggregates**: `SUM()`, `AVG()`, `COUNT()`, `MAX()`, `MIN()` (used with `OVER`)
    
- **Value functions**: `LAG()`, `LEAD()`, `FIRST_VALUE()`, `LAST_VALUE()`
    

---

## **5️⃣ Side-by-Side Comparison**

**Dataset**:

|product|price|
|---|---|
|Apple|2|
|Apple|3|
|Banana|1|
|Banana|2|

**Query**:

```sql
SELECT 
    product,
    price,
    SUM(price) OVER(PARTITION BY product) AS total_per_product
FROM Sales_Table;
```

**Result**:

|product|price|total_per_product|
|---|---|---|
|Apple|2|5|
|Apple|3|5|
|Banana|1|3|
|Banana|2|3|

---

💡 **In short**:

- `GROUP BY` → collapses rows into summaries.
    
- `OVER()` → calculates summaries **while keeping all rows intact**.
