Joints are used for extracing data for diffrent Tables or combining them. 

Types Of Joins:

__INNER JOINS__
An **INNER JOIN** combines rows from **two tables** **only** when there’s a matching ==_row_== values  in both tables based on a given condition.  like id.
```SQL
SELECT a.id, a.name, b.price
FROM Table_a AS a
INNER JOIN Table_b AS b
    ON a.id = b.id;
```

The highlighted portion is the matching data/row values that the both table must contain to be highlighted by inner joint clause 

__LEFT JOINS__

A **LEFT JOIN** combines matching rows off **two tables** **along with** All the row of table A, and discard the rows of Table B that does not contain in the table A, based on a given condition.  like id.

```SQL
SELECT s.sale_id, s.item_code, s.quantity, p.product_name
FROM Sales_Table AS s
LEFT JOIN Product_Info AS p
    ON s.item_code = p.code;



```

![[image.psd-8 1.png]]



RIGHT JOINS__

A **RIGHT JOIN** combines matching rows off **two tables** **along with** All the row of table A, and discard the rows of Table B that does not contain in the table A, based on a given condition.  like id.

```SQL
SELECT s.sale_id, s.item_code, s.quantity, p.product_name
FROM Sales_Table AS s
LEFT JOIN Product_Info AS p
    ON s.item_code = p.code;

```

__FULL JOIN / Union__
FULL JOIN is the join type that keeps **all rows from both tables**, whether they match or not.

```SQL
SELECT a.col1, a.col2, b.col3
FROM TableA AS a
FULL JOIN TableB AS b
    ON a.key = b.key;
```
UNION  MySQL do not suppot it so we use union clause

```SQL
SELECT *
FROM TableA a
LEFT JOIN TableB b ON a.key = b.key
UNION
SELECT *
FROM TableA a
RIGHT JOIN TableB b ON a.key = b.key;

```
